# 계열사 시너지 확장 매트릭스 — JB금융지주

> 마중(Majung) 엔진(3중 게이트·위임장·다국어 인상)을 JB금융지주 계열사로 확장하는
> 사업 논리와 기술 재사용 범위. 당기 구현 외 항목은 **로드맵** 으로 표기.

## 2. 계열사 확장 매트릭스

| 계열사 | 확장 대상 | 재사용 모듈 | 신규 필요 | 기대 효과 | 시점 |
|---|---|---|---|---|---|
| **광주은행** | 광주·전남 거주 베트남·중국 유학생, 계절 농업 E-9 | Gate A·B·C 전체, 위임장, 다국어 인상(vi/zh) | 광주은행 계좌 시드·한도계좌 필드, `fx_pair` 파라미터 추가만 | 호남권 외국인 신용 공백 해소, 전북은행과 고객 중복 최소화 | 로드맵 |
| **JB우리캐피탈** | 외국인 오토론·리스 신청 고객 | Gate C(AML), 다국어 안내(vi/zh), 가심사 엔진(`prescreen_refi` 재사용) | 캐피탈 상품 파라미터(금리·한도), 별도 세그먼트 시드 | 비은행 여신까지 신용 사다리 연장, 이륜차·소형차 할부 수요 흡수 | 로드맵 |
| **한패스 (JB 2대주주 15%)** | 한패스 앱 57만 외국인 사용자 중 전북은행 미거래 고객 | 송금 레일(`run_remittance`), 위임장, Gate B(FX 조건) | 한패스 API 인증, 수취 계좌 화이트리스트 확장 | 기존 57만 사용자 기반 즉각 온보딩, 한패스 상장(2026.3) 시너지 | 협의 중 |

## 3. 기술 재사용 원칙

- **동일 엔진, 다른 입구**: 계열사마다 새 `segment=` 파라미터와 시드 파일만 추가.
  게이트 로직·AML 임계값·위임장 서명 흐름은 수정 없이 재사용.
- **동결 구역 불변**: `orchestrator.py`·`rules.py` 불변. 계열사 확장은
  `activation.py` 또는 별도 `<company>_segment.py` 파일 추가로만 처리.
- **다국어 인상 확장**: `WELCOME` 객체에 `id`(인도네시아어) 등 키 추가만으로
  추가 국적 근로자 온보딩 가능.

|---|---|---|---|
| 광주은행 | 광주·전남 E-9·D-2 약 6만 명 | 전환 5% | 대환 이자 차익 + 송금 수수료 |
