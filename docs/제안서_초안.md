> [데모 스크린샷: 자동 송금 실행, 베트남어 통지, 철회 버튼]

**[한 줄 정의]** 마중은 급여 흐름 위에서 ① 환율이 유리할 때 자동으로 송금하고 ② 사채를 전북은행 대환으로 자동 안내하는 위임형 뱅킹 에이전트이다. 근로자와 유학생이 같은 엔진을 두 입구로 사용한다.

## 2. 문제 정의 (Slide 2)

> 발생 문제와 니즈, 대상, 중요성과 배경, KPI를 다룬다. As-Is 고객여정 그림을 삽입한다.

**[누구]** 두 세그먼트가 같은 진입로를 공유한다. ① 국내 체류 외국인 근로자 273만 명(역대 최대), E-9 30만 명 돌파, 1차 페르소나는 베트남 E-9 Tran Van Minh. ② 외국인 유학생 25만 3천 명(2025년, 비수도권 45.8%), 2차 페르소나는 중국 D-2 Wang Wei.

**[JB가 이미 문을 열었다]** JB금융그룹은 외국인 금융의 입구(가계좌), 채널(브라보코리아·라운지), 송금망(한패스), 신용 인프라를 이미 모두 구축했다. 전북·광주은행은 입국 직후 외국인을 위한 가계좌 시범사업을 2026년 3월 발표했다(91일 이상 장기체류·재외동포 대상, 생체인증 기반 한도제한 계좌).

**[그런데 문을 연 다음: As-Is 고객여정 그림]**
입국에서 외국인등록증 발급까지 최대 8주의 금융 공백이 발생한다. 그 사이 입국 전 브로커 사채(1,500만~2,000만 원, 연 20~40%)에 묶이고, 고금리 상환과 연체 시 "출입국 신고" 협박을 겪으며, 제도권은 "신용 이력 없음"으로 거절하여 다시 사채로 돌아간다.
> 부산 사례: 한 조직이 외국인 9,120명에게 162억 원을 빌려주고 연 최고 154%를 받았다.

**[문을 연 다음의 다섯 벽]**
1. **한도의 벽**: 가계좌와 한도계좌는 열렸으나, 신규 계좌 1일 송금 한도(30만 원)와 한도제한이 실생활을 막는다.
2. **이탈의 벽**: 일요영업점 대기 1~2시간, 한국어 전용 절차로 고객이 다른 채널로 빠져나간다.
3. **사채의 벽**: 5대 은행에 외국인 신용대출 상품이 없어 제도권 진입로가 막혀 있다.
4. **언어의 벽**: 브라보코리아 리뷰: "외국인용 앱이라면서 동의 문서가 전부 한국어다. 못 읽는 것에 서명하지 않는다."
5. **신용의 벽**: 신용 이력이 없어, 성실히 갚아도 다음 대출로 이어지지 않는다.

**[왜 지금]** 신한 SOL글로벌론은 "3개월 연속 급여 수령"을 대출 자격요건으로 삼아 급여계좌 선점 경쟁을 시작했다. 전북은행(외국인 신용대출 점유율 72%, 누적 24만 명)의 방어가 시급하다.

**[KPI]** 신규 모객 수가 아니라 LTV와 대환 전환율과 급여이체 전환율로 측정한다.

## 3. 제안 솔루션 개요 (Slide 3)

> 전체 구조, 서비스 구성, 시나리오 요약을 다룬다. 3중 게이트 구성도 그림을 삽입한다.

**[한 구조, 두 단계]**
- **Step 1. 위임 송금(유입 채널)**: 급여 감지, 환율 Rule 판단, 한도 내 자동 실행, 사기 자동 보류. 상호투표와 첫인상용.
- **Step 2. 신용 형성(핵심 수익원)**: 급여와 부채 분석, 사채 대환 가심사(연 약 250만 원 절약), 적금 연계. 실무진의 도입 판단이 이루어지는 지점.

**[포지셔닝]** 별도의 신규 앱이 아니다. 마중은 브라보코리아(JB의 외국인 슈퍼앱, 그릇) 안에서 동작하는 전환 엔진(Conversion Engine)이다. JB가 깐 인프라를 고객 한 사람에게 자동으로 잇는 활성화 레이어로, 기존 외국인 고객 누적 24만 명과 가계좌 모수를 대환과 적금으로 활성화한다.

**[아키텍처 그림: 3중 게이트]**
발화 또는 급여 입금 → LLM 의도 파싱(자연어만) → Gate A 위임장 검증 → Gate B Rule 한도·FX → Gate C AML → 실행, 모국어 통지, 철회.

**[제1원칙]** 송금 절차에서 LLM을 배제하여 할루시네이션에 따른 금융 사고를 원천 차단한다. 판정과 실행은 전부 결정적 코드가 수행하며, 이것이 환각과 책임 소재 질문의 정답이다.

| (유학생 Wang Wei) 한도계좌를 언제 정식계좌로 풀 수 있나 | 한도해제 코치(연속 급여 3개월 판정)와 재학 중 신용 형성 |

**[핵심 기능 = Agent 루프, 평가 용어 자구 그대로]**
- 큰 루프: **판단**(Gate A·B·C) → **행동**(실행·보류·알림) → **검증·개선**(사후 성과를 반영한 위임 임계값 조정)
- 세부 루프: **수집 → 검색 → 판단 → 생성 → 검증 → 후속액션**

**[MVP 구현 핵심 기능: 전부 동작, 자동 테스트 29건과 e2e 검증]**
① 위임장 발급(전자서명, 건별 통지, 철회권). ② 위임 송금 자동 실행. ③ 의심 거래 자동 보류와 STR 후보 큐 등록. ④ 사기 즉시 차단. ⑤ 대환 가심사 후 전북은행 회부. ⑥ (유학생) 등록금 위임 송금과 한도해제 코치. 유학생 등록금 송금은 근로자와 동일한 3중 게이트를 KRW/CNY로 재사용한다.

**[이 시스템이 하지 않는 것: 설계로 그은 경계]**
LLM이 송금을 실행하지 않는다. 에이전트가 대출을 승인하지 않는다(전북은행 회부만). 환율과 사기를 예측하지 않는다(결정적 Rule). 외부에 식별정보를 보내지 않는다. 위임 범위 밖을 임의로 실행하지 않는다. 단일 사고의 최대 손실은 위임 한도 내로 한정된다.

## 5. 데이터 및 기술 활용 (Slide 5)

> 활용 데이터, 처리 방식, 모델과 도구, 기술 제약과 해결을 다룬다.

**[활용 데이터]** 급여 입금, 송금 이력, 환율(7일 이동평균, KRW/VND와 KRW/CNY), 수취인, 사채와 대환 상품, 한도계좌 상태. MVP는 전부 모의·합성 데이터로 실 PII를 사용하지 않는다.

**[처리 방식]** 외부 LLM에는 식별정보를 보내지 않고(토큰화) 의도 파싱과 번역만 전달한다. 판정은 결정적 룰(FX, AML, 대환, 한도해제)이 수행한다. 전 행동을 감사로그에 적재한다.

**[모델·라이브러리·도구]** Python, FastAPI, SQLite, React(Vite), MCP SDK. 전부 허용형 라이선스(MIT, BSD, Public Domain)이다. LLM은 상용 API(식별정보 비전달, 학습 미사용 약관)이며 실행 권한이 없다. 활성화 레이어는 별도 모듈 `activation.py`로 송금 실행 경로와 분리한다.

**[기술 제약 → 해결]**
| 제약 | 해결 |
|---|---|
| 코어뱅킹 실연동 불가(망분리, 보안성 심의) | 모의 API e2e PoC, 실연동은 서비스화 로드맵 |
| LLM 할루시네이션과 책임 | 송금 절차에서 LLM 배제, 화이트리스트와 한도로 최대 손실 상한 |
| 환율 예측 불가 | 7일 평균 1% 이상 Rule(예측이 아님), 신선도 60초 정지 |
| 외국인 AML 위험 | Gate C 스코어링, STR 후보 큐, 의심 탐지를 자동 실행보다 앞단에 배치 |

## 6. 사용자 시나리오 / 유즈케이스 (Slide 6): 두 입구 한 사다리

> 주요 사용자와 행동 흐름. 유스케이스 다이어그램 그림을 삽입한다.

**[주요 사용자]**
- 근로자 Tran Van Minh(가명), 베트남 E-9. 입국 시 브로커 사채 1,500만 원, 한국어 미숙, 급여 일부를 모국으로 송금.
- 유학생 Wang Wei(가명), 중국 D-2. 가계좌로 입국, 한도계좌 단계, 등록금을 모국에서 송금받아 납부, 졸업 후 취업비자 전환을 준비.

**[근로자 행동 흐름: Step 1과 Step 2]**
1. (베트남어) 위임 설정 후 모국어 재확인과 전자서명.
2. 급여 입금 감지, 환율 +1.82%, 고객 개입 없이 자동 송금과 베트남어 통지.
3. 새벽 신규 수취인 480만 원, AML 점수 75, 자동 보류와 "정말 보내려는 분이 맞습니까?" 질문.
4. "OTP와 신분증을 보내면 대출" 수취인, 블랙리스트 기반 즉시 차단.
5. 에이전트: "사채 연 30%를 전북은행 연 13.59%로, 연 246만 원 절약" → 절약액과 월상환액과 연체 불이익을 동일 비중 모국어로 고지 → [전북은행에 신청], 최종 승인은 전북은행.

**[유학생 행동 흐름: 같은 엔진]**
1. (중국어) 등록금 위임 송금. 대학 화이트리스트 수취인, KRW/CNY +1.846%, 동일 3중 게이트 통과 후 등록금 송금과 중국어·한국어 통지.
2. 한도해제 코치. 연속 급여 3개월 충족으로 정식계좌 전환 가능 안내(자금 이동 없음).
3. 재학 중 신용 형성. 연속 급여와 정시 거래를 신용 이력 스냅샷으로 제시.
4. 졸업 전환 가심사. 졸업과 취업비자 전환 시 기존 대환 가심사 경로로 연결.

**[유스케이스 다이어그램 그림]** 외국인 근로자와 유학생과 관리자, 전북은행 심사엔진, 한패스 레일 액터에 위임 송금, 보류, 차단, 대환, 등록금 송금, 한도해제 유스케이스를 배치한다. 근로자와 유학생이 같은 위임장과 같은 3중 게이트를 공유한다.

## 7. 기대 효과 및 향후 확장성 (Slide 7): 논리로 닫기

> 정량과 정성 효과, 확장성. 여기서 핵심 개념어(신용 사다리)와 부서 설득과 클로징을 배치한다.

**[정량 효과: 기본 전환율 7% 가정]**
| | 보수 3% | 기본 7% | 낙관 12% |
|---|---|---|---|
| 대환 전환 잔액 | 297억 원 | **693억 원** | 1,188억 원 |
| 연 이자수익 | 28.5억 원 | **66.5억 원** | 113.9억 원 |

1인당 연 246만 원 절약(1,500만 원 × (30% − 13.59%)). 대면 모집인 비용과 고객획득비용(CAC)을 최소화하고 플랫폼 연계 수수료만 발생한다. 송금 수수료는 대환 이자수익의 약 2.5% 규모로, 유입은 송금에서, 핵심 수익은 대환에서 창출됨을 보여준다(693억 원·66.5억 원은 자체 추정).

**[정성 효과]** 계좌 하나가 주거래로, 다시 신용 사다리로 이어진다. 이 전환은 시스템이 강제한다(신규 계좌 직후 20영업일 타행 개설 제한, 외환 지정거래은행, 한도계좌의 급여 3개월 해제 조건). 사채와 환치기의 자금 흐름을 합법 레일로 흡수하여 AML 가시성을 개선한다.

**[JB 위상]** JB금융그룹 외국인 대출 잔액은 1조 269억 원으로 국내 최초 1조 원을 돌파했고(2026.2말), 그룹 점유율은 70% 이상이다(2026.3). 연말 1.3~1.5조 원, 장기 3조 원을 전망한다(회장 컨퍼런스콜). 마중은 이 성장축의 활성화 레이어이다.

**[실무진 우려 → 마중의 한 줄]**
| 부서 | 우려 | 마중의 답 |
|---|---|---|
| 외국인금융부 | "브라보코리아를 분산시키는 새 앱" | 브라보코리아의 전환 엔진, 신규 모객이 아닌 LTV |
| 디지털전략부 | "빅테크 채널 종속" | 본체는 자체 챗 UI, 외부는 선택권 |
| 여신심사부 | "자동 대환은 불완전판매" | 안내와 가심사까지, 최종 승인은 전북은행 심사엔진 |
| 준법감시 | "자연어 위임은 규제 사고" | 전자서명 강제, 건별 모국어 통지, 100% 철회권, AML 선차단 |

**[확장]** 베트남어와 중국어에서 17개국어로, 텍스트에서 음성으로, 근로자와 유학생에서 소상공인까지. 모의 API에서 실 API와 코어뱅킹 연동으로. 재학 이력 기반 외국인 신용평가(CB) 연계와 등록금 스테이블코인 정산은 Future Work 로드맵으로 분리한다(확정 서비스 단정 금지).

**[클로징 한 문장]** 우리가 못 파는 이유는 상품이 없어서가 아니라 고객이 우리를 못 찾아서이다. 마중은 점유율 72%, 누적 24만 명의 데이터 위에서, 모집인 없이 사채 고객을 전북은행 대환으로 데려오는 유입 채널이다.
