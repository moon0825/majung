## 슬라이드: 대환 메커니즘
"대환"을 정확히 정의한다. 사채를 대신 갚아주는 게 아니다. 송금으로 쌓은 기록으로 자격을 만들고, JB 신용대출로 갈아타는 것이다.

```
신규 도착자: 소득 기록 0 → 제도권 거절 → 사채 (신한은 "3개월 연속 급여"를 요구한다)
   ↓
1단계 자동송금이 3개월간 급여·거래 기록을 남긴다 → 내부등급 산출 가능
   ↓
가심사 게이트(결정적 코드): 비자 잔여 > 만기 + 완충 · DSR ≤ 0.40 · 다중채무 < 3 · 급여 연속성
   ↓
JB 신용대출 연 13.59% (실행과 최종 승인은 JB 심사엔진)
   ↓
대환금 사용처 추적: 등록 대부업자 우선. 불법 사채는 신고·수사 연계 트랙으로. [B, 파일럿 설계]
```
- 멘트: "직접 갚아주지 않는다. 기록으로 자격을 만들고, JB 대출로 갈아탄다. 출국·회수 리스크는 비자 잔여기간으로 게이트에서 거른다."
- 정직하게 둘 것: 대환금이 불법 사채 계좌로 가는 경우의 처리(에스크로·신고 연계)는 파일럿에서 설계한다. 지금 다 풀었다고 말하지 않는다.

## 슬라이드: 숫자는 하나만 단정한다
단정하는 숫자는 고객 절약액 하나다.
- **1인당 연 246만 원.** 1,500만 원을 사채 연 30%에서 JB 연 13.59%로 옮길 때의 이자 차액이다. 두 금리 모두 출처가 있다(사채 금리는 부산 적발 사건 등, JB 대환 금리는 상품 고시). 산식은 코드 테스트로 검증돼 있다.

단정하지 않는 숫자는 단정하지 않는다.
- 대환 잔액, 연 이자수익, 전환율은 내부 데이터가 없으면 확정할 수 없다. 추정치를 헤드라인으로 쓰지 않는다.
- 대신 구조를 보여준다. 깔때기: 외국인 고객 → 위임 동의 → 사채 보유 → 가심사 적격 → JB 본심사. **각 단계 전환율은 JB가 파일럿 실데이터로 채운다.**
- 수익 귀속: 이자수익·LTV·휴면 재활성은 JB로 간다. 단가·전환율을 JB 데이터 없이 추정하지 않으므로, 팀 자체 수익 추정도 제시하지 않는다.
