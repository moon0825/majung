## S5. 아키텍처 — 3중 게이트 (핵심 1장)
```
발화/이벤트 → LLM 의도파싱(자연어만) → ║Gate A 위임장│Gate B Rule·FX│Gate C AML║ → PASS만 실행 → 건별 모국어 통지+철회
```
- **제1원칙: LLM은 execute_remittance를 호출조차 못 한다.** 판정·실행은 전부 결정적 코드.
- 말하기: "환각이 나도 화이트리스트 밖 수취인·한도 초과는 코드가 거절합니다. 구조로 막습니다."

## S6. 라이브 시연 (무키 모드)
- ② **고객 개입 없는 자동송금**(새벽, 손·커서 없음) → ③ 보류+STR → ④ 블랙리스트 차단 → ⑤ 대환 4블록 동일고지 → 관리대시보드(게이트 트레이스·STR 큐·검증/개선)
- 말하기: "지금 화면에 제 손이 없습니다. 조건이 맞아 코드가 스스로 실행합니다."

## S7. 사업 — 1단계는 2단계의 깔때기
- **1단계 자동송금 = 신뢰·신용/소득 데이터 적립 → 2단계 대환 전환의 깔때기.**
- 단위경제: 1인 연 246만 절약(확정). 모수 6.6만(휴면 20만+고객 24만 중 사채 15%)에 **전환 7% 가정** → 대환 잔액·이자수익(자체 추정). NIM 9.59%p.
- 민감도(보수/기준/공격): 전환율 3/5/7/10% 시나리오로 제시 — "7% 한 점 베팅 아님."
- 말하기: "송금 수수료가 목적이 아닙니다. 사채에 묶인 고객을 JB 대환으로 전환하는 게 수익원입니다."

## S8. 【방어①】 전금법 — "접근매체 양도 아님" *(실무진 1순위 의심 선제 차단)*
- 마중은 비번·OTP·인증서를 **넘기지 않는다.** 고객이 사전 전자서명한 위임장(수취인·한도·조건·**유효기간·철회권**) 하에 **본인의 거래지시를 조건부 자동 집행** — 자동이체·펌뱅킹 약정과 동급.
- 판례상 처벌되는 '양도' = **"기간 제한 없는 배타적 이용권의 확정적 이전"**(대법원 2011도16167). 마중은 철회권·유효기간·범위 有 → 확정적 이전 아님.
- 말하기: "위임은 양도가 아닙니다. 언제든 거둘 수 있는 사전 거래지시입니다."

## S9. 【방어②】 망분리·외부 AI 보안 — 데이터 흐름
```
[고객 자연어 발화] ─→ LLM(의도 파싱)            ← 전송되는 것: 자연어뿐
                         │ intent(후보)
[계좌·실명·금액] ─── 결정적 코드 내부에서만 처리 ─→ 3중 게이트 → 실행
```
- LLM은 송금경로 밖 + **전송 데이터는 자연어 발화뿐**(계좌·실명·금액은 분리된 코드 내부). MCP는 도구별 권한 화이트리스트를 JB가 통제.
- **금융위 2024.8 「망분리 개선 로드맵」**(생성형 AI 활용 허용·SaaS 확대)이 허용하는 방향과 정렬.
- 말하기: "외부 AI가 우리 계좌를 보는 구조가 아닙니다. LLM엔 '말'만 갑니다."
