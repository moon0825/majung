# 마중(Majung) 본선 발표 (v5)

본선 7/4~5 현장 발표. 8분. 시연 라이브 3개, 나머지는 백업.
스파인은 민(Tran Van Minh) 한 사람. 톤 원칙: 숫자 먼저, 출처 붙이고, 형용사 뺀다. 비유는 주장이 아니라 이해를 돕는 그림으로만 쓴다.

**v5 재편 핵심**: JB가 밀던 그림(4대 성장축·플러그인 철학)을 뒤쪽 각주에서 앞단(전 3:30) 프레임으로 끌어올린다. 마중은 그 위에 고속도로를 깐다. 심사위원이 "우리가 밀던 그림을 마중이 고속도로로 깔아준다"를 3초 안에 머리에 그리게 한다. 감정은 양 끝(오프닝·클로징)에, 숫자는 가운데 블록에 둔다. 같은 슬라이드에 섞지 않는다.

## 슬라이드: 승 · 안전 처리 요약 (2:35–3:30)

시연 ①②를 라이브로 보인 뒤, 정지 1장으로 안전 구조를 못 박는다.

- 멘트: "판정과 실행은 결정적 코드가 합니다. LLM에는 자연어 발화만 가고, 계좌·실명·금액은 코드 안에만 있습니다. 금융위 2024년 8월 망분리 로드맵과 같은 방향입니다.² 마중이 의심 거래를 STR로 올리고, 담당자가 검토·보고·기각합니다. CDD와 CTR은 구조를 갖췄고, 임계값은 JB가 켭니다. 가중치도 임계값도 JB 룰북으로 바꿉니다."

게이트 트레이스(화면, 위임 한도 200만 이내 정상 거래):
```
Gate A  위임장 유효·범위·서명           PASS
Gate B  한도(≤200만)·환율 조건(FX 우위 +1.82%)   PASS
Gate C  수취인 AML 스크리닝             PASS
```

STR 워크플로 한 컷(8초, 풀 대시보드 아님):
```
STR 큐   | 거래                          | 스코어 | 담당자 | 상태
--------- ----------------------------- ------- ------- ------------------
후보     | 미등록 수취인 480만(위임 범위밖 | 75    | 심사역 | [검토] [보고] [기각]
          → AML 회부 → STR 보류)         |       |       |
```

- 배지(컴플): STR 「활성」(녹색) · CDD·CTR 「구조 갖춤 · JB가 활성화」(회색). 캡션: 「가중치·임계값은 JB 룰북으로 교체」
- ⚠️ 발표자 화법 규율: "CDD·CTR도 지금 작동 중"이라고 말하지 않는다. 코드에 임계값 판정이 없다. "구조를 갖췄고 JB가 켠다"로만 말한다.³ STR 큐 처리 워크플로(검토/보고/기각)는 화면 증거로 보이되 "JB 연동 파일럿 단계"로 범위를 한정한다.
- ⚠️ 게이트 순서 방어(리허설 필독): STR 큐의 480만은 **미등록 수취인(위임 범위 밖)** 거래다. 코드(orchestrator.py)는 위임 범위 밖 시도를 한도로 하드 거절하지 않고, `on_exception=hold_and_ask` 정책으로 AML(Gate C)에 회부한다. 한도·FX 자동집행은 **위임 범위 안** 거래에만 적용한다. AML이 스코어 75(위임밖30+신규25+고액20)로 STR 보류·큐 적재한다. 심사위원이 "한도가 200만인데 480만이 왜 실행 흐름에 있나"라고 물으면 "실행된 게 아닙니다. 위임 범위 밖이라 자동집행 대상이 아니고, AML로 올려 STR 보류했습니다. 위임 범위 안 거래만 한도·FX로 자동 판정합니다"로 답한다. 이 2-트랙(범위 안=한도 자동집행 / 범위 밖=AML 회부)이 방어의 핵심이다. 게이트 트레이스의 정상 거래(범위 안·한도 이내 PASS)와 STR 큐의 범위 밖 거래는 서로 다른 두 건이다.

- ⚠️ 낭독 메모: 점유율은 화면에 「70%대(최대 72%)」로 병기한다. 발표자도 구두로 "외국인 신용대출 70%대, 최대 72%로 1위"로 말한다. "항상 72%냐"는 질문에 방어면을 넓게 둔다.

## 슬라이드: 전 · 대환 정의 (시연⑤ 진입 전 자막 1줄로 못 박음, 4:45)

"대환"을 정확히 정의한다. 마중은 사채를 대신 갚지 않는다. 송금으로 쌓은 기록으로 자격을 만들고, JB 신용대출로 갈아탄다.

```
신규 도착자: 소득 기록 0 → 제도권 거절 → 사채
             (신한 SOL글로벌론은 "3개월 연속 급여"를 요구한다⁸)
   ↓
자동송금이 3개월간 급여·거래 기록을 남긴다 → 내부등급 산출 가능
   ↓
가심사 게이트(결정적 코드): 비자 잔여 > 만기 + 완충 · DSR ≤ 0.40 · 다중채무 < 3 · 급여 연속성
   ↓
```

- 백엔드 다운 시 오프라인 목 모드(healthRef=false)로 자동 전환. **고객 화면 카드(②자동송금·③보류·④차단·⑤대환)만 재현된다.** 관리자 대시보드의 STR 큐·감사로그·FX는 백엔드 응답에만 의존해, 오프라인이면 STR 큐가 빈 EmptyState로 뜬다. 게이트 판정 트레이스 표만 세션 로컬 state로 오프라인에서도 채워진다.
- ⚠️ 승 2:35 STR 큐 적재 장면은 오프라인 목으로 재현 불가. 이 컷만은 백엔드가 살아 있는 상태를 전제로 하거나, 녹화본을 별도 확보해 튼다.
- ③·④ 녹화 파일은 오프라인 목(고객 화면)으로 대체 가능하다. STR 큐 화면은 대체 불가.
- 고속도로 리빌 애니메이션 실패 시 「끊김」·「완성」 정적 2장으로 폴백.

## ⚠️ 화면 표시 리스크 (발표 전 점검)

- **[해소] 관리자 대시보드 BusinessValuePanel 폐기 수치.** BIZ.refiBalanceKrw·annualInterestKrw·refiConversionRate가 과거 693억·66.5억·7%였으나 커밋 fc87027에서 **504억·48.1억·7%(출처맵 기준값)로 교체 완료.** 이제 대시보드 탭을 열어도 폐기 수치가 뜨지 않는다. (초기 demo-qa 지적은 교체 전 상태 기준이었음.)
- **[해소] 고객 화면 GatePills CDD 단정 배지.** "CDD 고객확인 완료" → **"CDD 구조 갖춤"으로 교체(Cards.jsx).** 규제방어의 "구조 갖춤·JB 활성화" 프레임과 정합. CTR은 "비대상(계좌이체)" 유지(현금거래 대상이 아니라 사실 정확). 발표자도 "구조를 갖췄고 JB가 켠다"로 말한다.
- **STR 큐 상태 배지가 raw 'queued'로 노출된다.** AdminDashboard는 status==='pending'만 '심사대기'로 바꾸는데 실제 적재 status는 'queued'라, 화면 클로즈업 시 영문 'queued'가 그대로 보인다. 발표 지장은 작으나 클로즈업 촬영 시 인지한다.
- **월 상환액이 오프라인 목(509,723원)과 라이브 백엔드(509,683원)에서 40원 다르다.** 앵커 246만(2,461,500원)은 양쪽 정확히 일치한다. 폴백으로 오프라인·라이브를 섞으면 월 상환액 클로즈업이 미세하게 달라지니, 재촬영 시 한쪽으로 통일한다.
