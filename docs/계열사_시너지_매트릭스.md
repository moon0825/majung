# JB금융그룹 계열사 시너지 확장 매트릭스

> 마중의 위임형 에이전트 엔진(3중 게이트·모국어 대화·신용사다리)은 전북은행 단독이 아니라
> JB금융그룹 계열사 전체로 같은 구조를 확장할 수 있다.
> 추가 인프라를 새로 깔지 않아도 된다 — 같은 엔진을 얹기만 한다.

## 확장 매트릭스

| 계열사 | 주 고객 | 마중 엔진 적용 | 기대 시너지 |
|---|---|---|---|
| **전북은행** | E-9 외국인 근로자 | 위임 송금 + 대환 가심사 (현재 시연) | 외국인 신용대출 점유율 70%대 선점 유지·확장 |
| **광주은행** | 광주·전남 외국인 근로자(제조·조선) | 동일 위임엔진 광주 지역화 | JB그룹 외국인 대출 1조 기반 추가 지역 커버 |
| **JB우리캐피탈** | 외국인 할부금융·리스 수요 | 할부 조건 위임 설정(이율 기준, 상환 트리거) | 캐피탈 여신+은행 대환 연계 크로스셀 |
| **한패스** (JB 2대주주 ~15%) | 외국인 57만 명 송금 사용자 | 마중 위임 레일로 한패스 송금망 활용 | JB 전략투자 사업 시너지·외국인 금융 생태계 완결 |

## 확장 원칙

1. **엔진 공유**: 3중 게이트(Gate A/B/C) 코드는 계열사 공통. 언어팩·상품 파라미터만 교체.
2. **LLM 배제 원칙 유지**: 모든 계열사 적용에서도 LLM은 의도 파싱만, 실행은 결정적 코드.
3. **규제 레이어 분리**: 특금법 STR·CDD·CTR 컴플라이언스는 각 계열사 기존 시스템과 연계, 마중은 통제 계층만.
4. **한패스 레일 우선**: 송금 실행은 한패스 기존 외환 신고·한도 체계 위에서 동작. 신규 외환 인프라 불필요.

## 단계별 확장 로드맵 (제안)

```
파일럿: 전북은행 E-9 외국인 자동송금·대환 가심사 (현재)
   ↓ D+90
광주은행 지역화: 언어팩(베트남어·인도네시아어) + 광주 지역 수취인 화이트리스트
   ↓ D+180
JB우리캐피탈 할부 위임: 이율 기준 자동상환 트리거 + 연체 조기경보
   ↓ 한패스 상장 이후 협력
한패스 레일 통합 심화: 마중 위임장 ↔ 한패스 API 직접 연동, 해외 수취인 화이트리스트 공유
```

> 모든 수치(1조, 57만, 15%)는 공개 출처(머니투데이·블로터·뉴스핌) 기반. 상세: `docs/출처맵.md`.
