# 마중(Majung) — JB Fin:AI Challenge 본선 개발계획 (6/27 → 7/3)

> 본선: **7/4~5 (1박2일, 현장 발표·시연·Q&A)**. 작성 2026-06-27.
> 원칙: **7/3 동결(freeze)** 을 역산 마감으로 못박고, 점수가 갈리는 축(발표·Q&A·체감도)에 자원을 집중한다.

## 0. 한 줄 진단
마중은 **문제정의·규제준수가 만점급**(레포가 평가용어를 자구 그대로 정렬), 백엔드 기술도 견고(20/20 테스트, 3중 게이트 단일 실행경로). **등수를 가르는 잔여 리스크 3개**: (1) LLM이 룰 fallback이라 '진짜 AI 에이전트' 인상이 약할 수 있음, (2) 본선 라이브 발표/Q&A 준비 미완, (3) 검증·개선 루프가 정적 텍스트. → 아래 **집중 3가지**가 정확히 이 셋을 겨냥.

**실무진 체감 관점 최우선(`docs/실무진_공감_분석.md`):** ① 전금법 **"접근매체 양도 아님" 방어 논리**가 없음(실무진이 가장 먼저 의심할 위법성 — 문서 1장으로 선제 차단) ② **DSR 하드코딩 0.32**(여신담당 즉시 "가심사 아니다" — 호출부만 고치면 룰 무변경) ③ **CDD/CTR·외환 5만달러 연한도 부재**. 모든 보강은 **동결 구역(게이트/룰/송금경로) 무침범**.

## 1. 집중 3가지 (Top 3)
1. **[체감도]** 관리대시보드를 'JB 컴플라이언스 담당자가 그대로 쓰는 한 화면'으로 마감 + '검증→개선' 폐루프를 `audit_log` 실데이터로 1회 작동(읽기전용 `GET /metrics/loop`). 평가 라벨(판단→행동→검증/개선)이 슬로건이 아니라 **실제로 도는 루프**임을 한 화면으로 증명. *(송금 경로 불변, 표시·읽기 레이어)*
2. **[기술/Agentic]** `llm.py parse_intent`를 Claude(claude-opus-4-8) 실연동으로 교체 — **ParsedIntent 스키마·3중 게이트 100% 불변**, API키 없으면 정규식 fallback 유지. 라이브 Q&A에서 고정 10문장 밖 임의 베트남어를 즉석 파싱하되 실행은 게이트가 막는 장면으로 **"제1원칙: LLM은 통역, 금고 열쇠는 결정적 코드"** 를 말이 아닌 데모로 증명. *(동결 구역 인접·외부키 → 사람이 직접, 야간 자동 금지)*
3. **[발표/Q&A]** 본선 라이브 발표덱(페르소나 15초 오프닝→3중게이트 단일 다이어그램→1단계 송금이 2단계 대환의 깔때기인 사업퍼널→규제 내장 착지)과 킬러질문 6~8종 30초 방어카드(`본선_QnA_방어카드.md`)를 7/3까지 완성·동결. '②고객 개입 없는 자동송금'을 무키 오프라인 모드로 **최소 5회 리허설.**

## 3. 날짜별 계획 (각 날짜: 사람 필수 ↔ 야간 루틴 자동 분리)

- **야간자동:** NIGHTLY_BACKLOG 반영·PROGRESS_LOG, 출처맵/README 추정·확정 수치 분리 일관성, 접근성 1차(aria-label·포커스).

- **야간자동:** Q&A 방어카드 markdown 보강, 출처/법령 인용 정확성 점검, 접근성·색대비 2차.
