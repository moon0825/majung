# 금융권 공식 산출물 표준 어체 (fin-biz-tone)

이 프로젝트의 모든 한국어 산출물은 이 어체만 사용한다. 독자는 은행 경영진과 공모전 심사위원이다.

## 1. 기본 톤

- 공식적이고 단단한 비즈니스 한국어. 직역투 금지.
- 고객 채팅 버블(B2C 폰 화면 안 대사)만 부드러운 존댓말을 허용한다. 그 외 모든 영역(문서, 관리자 화면, 다이어그램, 자막)은 격식 있는 비즈니스 문장으로 작성한다. 단, 어디에서든 은어와 em dash는 금지.

## 2. 개발 은어 순화표 (필수 변환)

| 은어 | 순화 |
|---|---|
| 1막, 2막 | 1단계, 2단계 또는 Step 1, Step 2 |
| 본진 | 핵심 수익원, 핵심 비즈니스 |
| (점수·시간) 컷 | 임계치, 기준점, 초과 시 정지 |
| 깔때기 | 유입 채널 |
| 무기화 | 전략적 활용 |
| 해자 | 진입 장벽, 경쟁 우위 |
| 잠식 | 트래픽 분산, 고객 이탈 |
| 풀가동, 올인 | 전면 운영, 집중 |
| 무개입 (UI 라벨로 단독 사용 시) | 고객 개입 없이 자동 실행 |

## 3. 개조식의 올바른 정의 (이것을 틀리면 전부 틀린다)

개조식은 단어를 생략해 욱여넣는 것이 아니다. 주어와 목적어의 관계가 명확히 드러나는 완전한 구나 절 단위를 유지하되, 문장의 끝맺음만 명사형(~함, ~지원, ~구축, ~배제, ~확보)으로 깔끔하게 처리한다.

- 나쁜 예: "급여 트리거 + 환율 Rule 자동 송금"
- 좋은 예: "급여 입금 트리거와 목표 환율 도달 조건에 기반한 자동 송금 실행"
- 나쁜 예: "3중 게이트 자동 보류·차단 (AML)"
- 좋은 예: "3중 게이트 기반 자동 보류 및 자금세탁방지(AML) 선차단"

## 4. 정보의 위계: 가운뎃점 금지, 인과로 연결

여러 개념을 나열할 때 가운뎃점(·)으로 욱여넣지 않는다. 논리적인 인과나 목적 관계("A를 통해 B를 달성함", "A를 적용하여 B를 차단함")로 자연스럽게 풀어낸다.

- 나쁜 예: "전자서명·건별통지·철회권 + AML 선차단"
- 좋은 예: "전자서명 강제, 건별 모국어 통지 및 100% 철회권 보장, AML 기반 선차단 로직 탑재"

허용 예외: 고유명사 병렬(전북은행·브라보코리아·한패스), 두 단어짜리 쌍(한도·환율, 보류·차단).

## 5. em dash(—) 전면 금지

본문, 제목, 표, 코드 주석, UI 문자열, 자막, 다이어그램 라벨 등 모든 곳에서 절대 사용하지 않는다. 콜론, 쉼표, 괄호로 대체한다. 흐름 표기의 화살표(→)는 도식과 단계 서술에 한해 허용한다.

## 6. 괄호 영어 병기 규칙

- 일반 단어에 영어를 덧붙이지 않는다: "구(Phrase)", "절(Clause)" 같은 표기는 금지.
- 금융과 기술의 표준 용어만 허용: LTV, CAC, NIM, AML, STR, PoC, JTBD, Lock-in, Capping, Hallucination, Explainability, STT 등. 첫 등장 시 1회만 병기하고 이후 한국어만 사용.

## 7. 부정형 슬로건의 재서술

무엇을 안 하는지만 단정하지 말고, 그것으로 무엇을 지키는지 목적을 함께 서술한다.

- 변환 전: "LLM은 송금 실행 경로에 없다"
- 변환 후: "송금 절차에서 LLM을 배제하여 할루시네이션에 따른 금융 사고를 원천 차단함"
- 변환 전: "에이전트는 대출을 승인하지 않는다"
- 변환 후: "대출 최종 승인 권한은 전북은행 고유 심사엔진에 위임하고, 에이전트는 가심사와 안내까지만 수행함"

## 8. 표제와 라벨

- 헤드라인은 결론이 담긴 완전한 구로 작성: "유입은 자동 송금으로, 핵심 수익은 사채 대환에서 창출"
- 섹션 라벨은 대괄호로 위계 표시: [현황], [위협], [전략], [제1원칙], [책임 경계]
- 평가 핵심 용어(판단, 행동, 검증·개선 / 수집, 검색, 판단, 생성, 검증, 후속액션 / Gate A·B·C)는 단어 자체를 보존하면서 자연스러운 문장 안에 배치: "데이터 수집 → 조건 검색 → 룰 기반 판단 → 모국어 메시지 생성 → 사용자 검증 → 후속 대환 액션"

## 9. 유지되는 기존 원칙 (korean-proposal 계승)

- 모든 수치에 출처 또는 계산식 명기. 빈칸 금지.
- 부정 병렬("X가 아니라 Y"), 삼단 카운트다운, 표지판 결론("결론적으로"), 이중 피동, 과장 어휘(획기적, 압도적) 금지.
- 두괄식 배치, 의미와 수치와 고유명사 100% 보존.

## 10. Before → After 표준 사례

| "2막. 사채 대환 (수익원)" | "Step 2. 사채 대환을 통한 신용 형성 (수익 창출)" |
