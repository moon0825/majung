# 노션 MCP 작성 함정 (jb-cloud-ops reference)

> 마중 산출물을 노션에 정리하거나 본선 자료를 노션 페이지로 공유할 때 쓰는 함정 모음. 노션 MCP는 줄바꿈과 이스케이프, 다국어 글자, 구조화 블록에서 조용히 깨진다. 깨진 결과가 심사 자료에 남으면 완성도 게이트에서 걸린다. 쓰기 전에 본다.

## 1. 줄바꿈: insert_content와 replace_content가 다르다

- `insert_content` 계열은 `\n`을 줄바꿈으로 풀지 않고 리터럴 `n`으로 처리한다. `"1단계\n2단계"`를 넣으면 화면에 `1단계n2단계`로 박힌다.
- `replace_content` 계열은 실제 줄바꿈을 그대로 쓴다.
- 처방: 줄바꿈이 들어가는 본문은 `\n` 이스케이프로 조립하지 말고, 실제 줄바꿈이 든 문자열을 그대로 넘긴다. insert가 꼭 필요하면 줄 단위로 쪼개 블록을 여러 번 넣는다.

Why: `\n`이 `n`으로 굳으면 표제와 본문이 한 줄로 뭉치고, 뒤늦게 발견하면 페이지를 다시 짜야 한다.

## 2. 한글은 직접, \u 이스케이프 금지

- `notion-create-pages`와 `replace_content`에는 한글을 직접 쓰고 실제 줄바꿈을 쓴다.
- `\uXXXX` 유니코드 이스케이프로 한글을 끼우지 않는다. 코드포인트 한 자리만 틀려도 다른 글자가 되고, 검수에서 눈에 잘 안 띈다.
- 처방: 본문은 사람이 읽는 한글 평문 그대로 넘긴다. 자동 생성 코드가 이스케이프를 끼우면 출력 전에 평문으로 되돌린다.

Why: 이스케이프 오타는 빌드도 통과하고 미리보기도 통과한 채 본문 한 글자만 바꿔 둔다. 평문으로 쓰면 그 위험이 사라진다.

## 3. 깨진 간체 점검

- 인코딩이 어긋나면 간체자가 비슷하지만 틀린 글자로 굳는다. 대표 예로 환율은 `汇率`인데 `环率`로 깨진다.
- 처방: 중국어가 들어간 페이지는 작성 후 깨진 글자를 grep으로 훑어 고친다.
  ```bash
  grep -nP '环率' <파일 또는 내보낸 본문>
  ```
- 알려진 교정쌍은 발견 즉시 아래에 한 줄씩 늘린다.
  - 环率 → 汇率 (환율)

## 4. table과 callout: XML 대신 구조화 invocation

- `<table>`, `<callout>` 같은 블록을 따옴표가 든 XML 문자열로 직접 넘기면 따옴표 이스케이프가 쉽게 깨져 블록이 통째로 텍스트로 떨어진다.
- 처방: 표와 콜아웃은 MCP의 구조화 인자(블록 객체)로 넣는다. 원시 XML 문자열로 조립하지 않는다. 셀 안의 따옴표와 줄바꿈도 구조화 인자 안에서 다룬다.

Why: XML 문자열 조립은 한 군데만 따옴표가 어긋나도 블록 전체가 평문으로 무너진다. 구조화 인자는 이스케이프를 도구가 책임진다.

## 5. 블록 스펙 요지

- 콜아웃: 아이콘(이모지) 한 개와 본문 한 단락. 강조 박스 용도다. 본문은 평문 한글, 줄바꿈은 실제 줄바꿈으로 넣는다.
- 토글: 제목 한 줄과 접히는 자식 블록. 긴 보강 설명을 접어 둘 때 쓴다. 자식은 별도 블록으로 넣고 제목 줄에 본문을 욱여넣지 않는다.
- 표: 머리행과 데이터행을 행 단위 구조로 넣는다. 셀 안 줄바꿈과 따옴표는 구조화 인자로만 넣어 4절의 이스케이프 사고를 피한다.

이 문서는 사고가 한 번 더 잡힐 때마다 교정쌍과 함정을 한 줄씩 늘린다.
