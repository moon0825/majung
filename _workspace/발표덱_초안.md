> "제도권 은행의 대답: 신용 이력 부족으로 대출 불가."

- 발화 포인트: "전북은행이 이미 외국인 신용대출 점유율 72%를 보유한 유일한 은행이기에, 이 문제를 풀 수 있습니다."

## [S2] 문제 정의 (2분)

**슬라이드 2A — 시장 규모**
| 수치 | 출처 |
|---|---|
| 체류 외국인 273만 명(역대 최대) | 법무부 2025.7 |
| E-9 비자 30만 명 돌파 | Korea Herald |
| 소득 중 송금 비중 56.5% | 통계청·법무부 2023 |
| 5년 누적 송금 25조 원 | 한국은행 국감자료 2025.10 |

**슬라이드 2B — 3중 페인포인트**
1. **언어 장벽**: 금융 서류 전부 한국어 → 불완전판매·서명 오류
2. **사채 의존**: 브로커 사채 1,500만~2,000만 원 / 연 20~40% (서울경제TV 2026.3)
3. **신용 공백**: 5대 은행 외국인 신용대출 전무 (머니투데이) → 신용 이력 미형성 악순환

**슬라이드 2C — 왜 지금?**
- 전북은행 외국인 대출 1조 돌파, 누적 24만 명 (서울경제TV 2026.3)
- 김기홍 회장: "새로운 성장축" · 2026 연말 1.3~1.5조 목표 (블로터 1Q26 컨콜)
- 한패스 57만 명 · 2026.3 상장 → 경쟁 격화 전 선점 시점

## [S3] 솔루션 — 마중 (4분, 라이브 데모 병행)

**슬라이드 3A — 아키텍처 1줄 요약**
> "LLM은 의도 파악과 모국어 안내에만 쓰고, 돈이 움직이는 경로에서는 완전히 배제합니다."

핵심 3중 게이트:
- Gate A: 위임장 검증 (Mandate Validation) — LLM 개입 없음
- Gate B: 규칙 한도·환율 조건 판정 — LLM 개입 없음
- Gate C: 수취인 화이트리스트 + AML 스크리닝 — LLM 개입 없음

**슬라이드 3B — 데모 시나리오 요약 (라이브 시연 진행 중 표시)**
| 시나리오 | 핵심 메시지 |
|---|---|
| ① 위임 설정 | 베트남어 한 문장 → 위임장 생성 → 전자서명 (언어 장벽 해결) |
| ② 자동 송금 (고객 개입 없음) | 새벽 2시, 잠든 사이 조건 충족 → 자동 실행 → 모국어 통지 |
| ③ 이상 보류 | 새 수취인 + 새벽 패턴 → 즉시 보류 → STR 대기열 적재 |
| ④ 사기 차단 | OTP 요구 패턴 → 하드컷 차단 (블랙리스트 Score 100) |
| ⑤ 대환 제안 | 연 30% 사채 → JB 13.59% 대환 → 연 246만 원 절약 투명 고지 |

**슬라이드 3C — 투명성 원칙**
- 절약 이자와 갚아야 할 원리금을 동일 크기로 고지 (불완전판매 차단)
- 가심사만 제공, 최종 승인은 JB 심사엔진 결정
- 철회권: 언제든 단 한 번의 메시지로 위임 취소

## [S4] 검증 & 규제 (1.5분)

**슬라이드 4A — 규제 준수**
- 특금법 STR·CDD·CTR 의무 이행 (KoFIU)
- 외국인 거주자 무증빙 송금 연 5만 달러 한도 준수 (은행연합회)
- 신규 계좌 1일 30만 원 한도·한도계좌 해제 조건 자동 안내

**슬라이드 4B — 기술 검증 포인트**
- 게이트 판정: 단일 실행 경로 보장 (lock/unlock 경합 없음)
- 오프라인 폴백: 백엔드 다운 시 모의 데이터로 UI 유지
- 테스트: 백엔드 pytest 29개 전통과

## [S6] 임원 논리 착지 (1분)

**슬라이드 6A — 왜 전북은행인가**
> "시중 5대 은행이 외국인 신용대출을 하지 않는 동안, 전북은행만이 72%의 점유율로 24만 명을 이미 고객으로 품었습니다. 마중은 이 인프라 위에서만 작동합니다."

**슬라이드 6B — 실행 리스크와 통제**
| 리스크 | 통제 방안 |
|---|---|
| AML 오탐 | Gate C Score 임계 40(경고)/70(STR) 이중 컷, 담당자 검토 대기열 |
