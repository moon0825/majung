## 위험한 모순 (즉시 보정 필요)

### 1. 특금법 CDD/CTR 구두 방어와 코드 불일치
- **위치:** Q&A 방어카드 Q3·Q10 및 본선_리서치 §4-4에서 "전신송금 CDD 100만·CTR 1천만"을 방어 근거로 제시
- **실제:** rules.py·orchestrator.py·mcp_tools.py 어디에도 CDD(100만 임계값 트리거)·CTR(1천만 임계값 트리거) 로직이 없음. 컴플라이언스 심사위원이 "CDD는 어디서 작동합니까?" 물으면 코드로 증명 불가
- **위험도:** 높음 — "말이 아니라 구현으로 증명"을 슬로건으로 쓰는 발표에서 코드와 불일치한 규제 주장은 신뢰 전체를 훼손
- **처치:** 구두 답변을 "CDD·CTR 의무는 JB 기존 컴플라이언스가 수행하고, 마중 에이전트는 STR 자동 회부와 보류·모국어 확인만 추가함"으로 즉시 수정. 방어카드 Q3·Q10에서 CDD 100만·CTR 1천만을 자체 구현 근거로 쓰는 부분 삭제 또는 "JB 레일 의존"으로 수정

### 2. 발표덱 클로징 "사채는 제도권으로 넘어갑니다" — 장래 이익 단정 소지
- **위치:** 발표덱 클로징(88번 줄) "잠든 사이 송금이 끝나고, 사채는 제도권으로 넘어갑니다."
- **문제:** 대환은 "가심사 → JB 최종 승인" 경로이며 JB 거절 가능. "넘어갑니다"는 현재 시제 단정으로 금소법상 부당권유(장래 이익 단정) 해석 위험. 동일 발표덱 전막에서 "가심사이며 승인 아님, 최종은 JB" 워터마크를 명시하라고 지시하면서 클로징에서 단정하는 내부 모순
- **처치:** "JB가 승인하면 사채에서 제도권으로 옮길 수 있습니다" 또는 "그 길을 만드는 것이 마중입니다"로 수정

### 3. 가심사 화면 [B] 워터마크 미완성 상태
- **위치:** 발표덱 전막 시연⑤ 자막 "가심사이며 승인 아님, 최종은 JB" 워터마크 [B] 표기
- **문제:** [B]는 "데모·코드 반영 필요" 미완성 항목. 본선 시연에서 워터마크 없이 "3개월 뒤 급여 3회 기록으로 가심사 적격" 화면만 뜨면 승인처럼 보일 수 있음 — 금소법 불완전판매 리스크
- **처치:** 본선 전 시연⑤ 화면에 "이는 가심사이며 최종 승인은 JB 심사엔진이 결정합니다" 워터마크 또는 배너 반드시 구현

## 추가 보강 제안

### A. Q3 AML 방어 보강 (structuring 미탐지 선제 공개)
- structuring·just_below_threshold 필드가 코드에 있으나 탐지 로직이 없음(항상 False). 컴플라이언스 담당자가 이를 발견할 경우 역공 위험
- 제안: "구조화 탐지 룰은 현 프로토타입에서 플래그 구조만 구현됐고, 실서비스에서는 JB 룰북 파라미터로 교체합니다"로 선제 언급

### B. late_night=0 가중치 설명 보강
- rules.py AML_WEIGHTS late_night=0은 오탐 방지를 위한 의도적 설계이나 "00~05시 거래를 AML이 아예 안 보나?"로 오해 가능
- 제안: Q3 답변 또는 컴플라이언스 백업 슬라이드에 "심야 가산 0이되 late_night 플래그 기록→STR 후검토 추적 가능" 한 줄 추가

### C. 전자서명 "공인전자서명 아님" 위험 사전 관리
- Q1 방어카드에서 "전자서명법상 효력 있는 수단으로 결합합니다"라고 하나 현 코드는 esign_hash 16자 절단(실무진_공감_분석 §1-④) — 본인확인 수단 연동 없음
- 발표덱 B2 백업에 "전자서명 본인확인 수단 연동 로드맵" 슬라이드 준비 필요(방어카드 하단 7/1 예정으로 이미 지정됨)
- 현 시연에서 esign 관련 기능 데모 시 "프로토타입 단계, 실서비스는 JB 채택 인증 수단 연동" 고지 필수

### D. DSR 하드코딩 0.32 노출 위험
- mcp_tools.py:267 dsr=0.32 모의값이 시연 중 노출될 경우 "심사를 조작한 것 아닌가?" 지적 가능
- 실무진_공감_분석 §1-③에서 최대 약점으로 명시됨
- 제안: 시연⑤에서 DSR 입력값을 화면에 표시하거나, "현 프로토타입은 모의 DSR, 실데이터 연동은 JB API 호출 구조로 대체 예정" 자막 추가

### E. AP2 매핑 표현 정확화
- 방어카드 Q11 "글로벌 표준"은 과장 가능성. AP2는 Google 단독 제안 프레임워크로 IETF·ISO 등 공식 표준 기구 채택 아님(미검증)
- 현재 방어카드에 "수렴 중"으로 이미 기재되어 있으므로 구두 답변도 동일하게 "수렴 중"으로 일관성 유지

### F. "백만 명" 오프닝 수치 확정 필요
- 발표덱 오프닝 "한국에서 일하는 외국인 노동자, 백만 명" ⚠️ 주석 있음 — 본선 전 확정 필요
- 외국인 취업자(통계청 2025 기준 약 92만) 또는 E-9 약 32만 중 선택·출처 병기 미완
